Approach

Simplicity

HTML Structure

Keep your HTML structure as simple as possible. Use HTML tag for content markup in first place without thinking about presentational layers that may be needed in the future.

Minimize number of wrappers that are needed only for styling purposes. Consider using pseudo elements ::after and ::before to move all decorative code from HTML to CSS.

Do you really think that UL needs its own extra wrapping div? Well it doesn't!

Classes in HTML

Do not overload HTML tags with long chains of CSS classes. If you are following component approach there shouldn't be such case in your code, when you need to apply more than 3 classes to single HTML tag. All components can be defined by applying 2 classes - component name class and component modifier (if default component needs modification).

In some cases extra helper class needs to be applied to component. But for sake of readability and code simplicity it is better to minimize using this helper classes as well and move all presentation to CSS layer.

Selector prefixes

While creating multi-level components follow namespace approach rather than using prefixes. For instance you're building some simple article block that has title, description and thumbnail. Define a top level class .article and use it as a namespace for styling nested elements. Look at this SASS example:


            .article
              .....
              .thumbnail
                ....
              .title
                ....
              .description
          

            .article
              ....
              .article-thumbnail
                ....
              .article-title
                ....
              .article-description

Prefixes look pretty obsolete here. If you always define .title and .thumbnail inside namespace, you'll never need to prefix them to be sure that you only affect titles and thumbnails inside article.

Do not user prefixes for nested elements. Style them using namespaces only.
